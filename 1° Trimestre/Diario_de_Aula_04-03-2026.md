## Quadro de Técnicas de Levantamento de Dados

| Técnica | Descrição | Quando Usar | Vantagens | Limitações |
|---|---|---|---|---|
|Entrevista|Técnica onde o analísta de requisitos e o cliente se encontram pessoalmente para tirar dúvidas do projeto|Utilizada no início de projetos para conhecer melhor o cliente e suas ideias.|Possibilita o cliente expor suas ideias com mais naturalidade|Nem sempre o cliente estará disponível para encontros|
|Análise de Documentos|Análise de documentos já existentes da empresa, como sistemas antigos, relatórios, entre outros|Quando existem documentos que podem ser aproveitados para analisar como a empresa funciona|Facilita a identificação de requisitos e não necessita realizar encontros com o cliente para extrair informações|Os documentos podem estar desatualizados em relação à forma como a empresa trabalha na atualidade|
|Brainstorming|Técnica realizada com um grupo selecionado de membros do projeto para exporem suas ideias em uma ou mais reuniões|Usada quando há necessidade de levantar muitas ideias e possíveis soluções para problemas sem respostas|Pode ser muito recompensadora com novas ideias proveitosas para o sistema|Não há garantia que as ideias propostas terão utilidade no desenvolvimento|
|JAD|Técnica colaborativa entre usuários, desenvolvedores e outros membros no levantamento dos requisitos e resolução de problemas|Quando é necessário alinhar o pensamento/conhecimento entre os envolvidos no projeto|Os usuários se sentem mais inclusos no projeto e a definição dos requisitos é melhorada|Nem semepre será possível reunir todos os membros necessários para a discussão do projeto|
|Prototipagem|Desenvolvimento de parte do produto final para apresentar aos clientes|Quando se sente a necessidade de ter uma aceitação do cliente a respeito do rumo que o produto está tomando|Validação precisa sobre a prototipação enviada ao cliente, indicando se pode ou não avançar para outra etápa do desenvolvimento|Pode demandar muito esforço e tempo para ter aceitação do cliente| 
|Estudo Etnográfico|Participação do analísta de requisitos no ambiente em que o produto será utilizado|Quando há um grande prazo para desenvolver o sistema e quando surge a necessidade de compreender a forma como os usuários irão fazer uso dele.|Podem ser encontrados requisitos não percebidos previamente em outros métodos de levantamento de requisitos que não utilizam o ambiente de trabalho como referência|Demanda muito tempo para incluir e perceber as necessidades que o ambiente de trabalho demanda para o sistema|
